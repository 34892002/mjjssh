# 功能开发计划

本计划记录已确认的功能范围、验收标准和实施进度。状态使用：`待开始`、`进行中`、`已完成`、`阻塞`。

## 一、终端体验

### 1. 断线重连

状态：`已完成`

目标：SSH 连接断开后保留现有页签和终端实例，复用既有的连接状态窗口完成手动重连。

验收标准：

- 远端关闭连接、网络中断或终端不可恢复时，终端明确显示断线原因并停止接收输入。
- 保留当前页签，不创建新窗口或新页签。
- 自动打开既有的 `ConnectionDialog`，以错误状态展示断线原因。
- 点击“重新连接”时，在同一 `sessionId` 上重新建立 SSH 连接，连接成功后恢复当前终端实例和输入能力。
- 重连失败时仍在同一个连接状态窗口内显示失败原因，可继续重试或关闭页签。
- 首版仅提供手动重连；自动重连策略另行设计，避免网络抖动导致意外连接循环。

### 2. 终端搜索

状态：`已完成`

验收标准：

- 基于已加载的 xterm `SearchAddon` 提供搜索工具栏。
- 支持关键字输入、上一个/下一个匹配、大小写敏感、正则表达式和关闭。
- 提供 `Ctrl+F` 打开搜索、`Enter` / `Shift+Enter` 跳转、`Escape` 关闭。
- 搜索工具栏不影响终端网格尺寸；打开、关闭和切换页签后终端可正确重新适配。

## 二、密钥管理

### 3. 在应用内生成 SSH 密钥对

状态：`已完成`

验收标准：

- 密钥管理提供“生成密钥”入口，默认生成 Ed25519。
- 支持 Ed25519 与 RSA；RSA 提供安全默认位数与可选位数。
- 首版生成无 passphrase 的标准 OpenSSH 私钥，确保生成后能立即用于现有连接链路。
- 私钥 passphrase 与连接时解锁会作为后续子任务实现；在支持前不展示不可用的 passphrase 输入。
- 生成后将私钥安全写入现有 Vault，显示并支持一键复制公钥。
- 新生成的密钥可立即用于创建或编辑主机。

## 三、诊断与隐私

### 4. 本地崩溃报告和诊断包导出

状态：`已完成`

验收标准：

- Rust panic、未处理前端异常和未处理 Promise rejection 写入本地崩溃报告。
- 默认不自动上传任何诊断数据。
- 设置页提供用户主动触发的诊断包导出。
- 诊断包包含应用/系统版本、专门设计为可导出的安全日志、非敏感配置摘要和本地崩溃报告。
- 安全日志必须从源头限制为运行元数据；不依赖对任意原始日志的通用正则脱敏。
- 默认不导出原始 `ai.log`；AI 输入输出、命令、用途和结果摘要均不进入诊断包。
- 必须排除密码、私钥、同步密码、Token、API Key、完整终端输入输出和远程文件内容。
- 用户可在导出前查看将包含的类别与隐私说明，并进行二次确认；提示内容必须明确说明：导出内容虽已进行初步隐私保护，仍可能因异常消息、调用栈或第三方库输出包含个人或敏感信息，用户应仅向信任的支持人员提供该文件。

## 四、主机身份验证

### 5. 自定义 `known_hosts` 与严格主机指纹校验

状态：`已完成`

验收标准：

- 应用在 `data/known_hosts.json` 单独保存受信任的主机密钥，不写入 Vault 或云同步数据。
- 信任记录以规范化后的 `host:port` 为键，保存算法、`SHA256` 指纹和确认时间。
- 首次连接在 SSH 认证前展示算法和指纹，用户必须通过显式“确认并信任”操作后才能继续。
- 已信任主机的服务端密钥与记录不一致时，必须阻止连接；连接窗口的指纹图标和进度状态显示为红色，并展示旧、新指纹。
- 重试或刷新主机信息均复用同一严格校验逻辑，不允许绕过主机身份验证。
- 首版不提供在指纹变更告警中直接覆盖信任记录的入口；用户需经独立的受控流程处理变更。

## 五、构建与发布

### 6. GitHub Actions Windows 构建

状态：`已完成`

验收标准：

- Pull Request 与主分支推送执行前端构建、Rust 格式/检查和测试。
- 使用依赖缓存缩短 Windows 构建时间。
- 推送版本 tag 时构建 Tauri Windows 安装包，并上传为 GitHub Release 附件。
- 安装包包含 WebView2 bootstrapper 配置，不引入自动更新机制或发布签名。
